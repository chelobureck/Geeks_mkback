# Geeks Education Platform

Платформа для управления образовательным процессом с ролевой моделью доступа (Студент, Преподаватель, Куратор).

## Архитектура
- `apps/`: Все приложения проекта (users, education).
- `core/`: Настройки проекта.
- `static/`, `media/`: Статические и медиа файлы в корне.
- `.env`: Конфигурация через переменные окружения.
- `sqlite3`: База данных по умолчанию.

## Технологии
- Django REST Framework (ViewSet, DefaultRouter)
- drf-yasg (Swagger)
- django-jazzmin (Admin Panel)
- Docker & Docker Compose

## Как запустить

### Локально
1. Создайте и активируйте виртуальное окружение: `python -m venv .venv && source .venv/bin/activate`
2. Установите зависимости: `pip install -r requirements.txt`
3. Настройте `.env` файл (укажите свои данные для SMTP).
4. Запустите скрипт: `sh run.sh` (или вручную миграции и сервер).

### Через Docker
1. Выполните команду: `docker-compose up --build`

## Роли и Права доступа

### 1. Студент (STUDENT)
- Видит только свои группы.
- Видит только свой табель (оценки).
- Видит свое расписание.

### 2. Преподаватель (TEACHER)
- Видит группы, которые он ведет.
- Может выставлять и менять оценки в своих группах.
- Видит расписание своих групп.

### 3. Куратор / Завуч (CURATOR)
- Полный доступ ко всему.
- Может добавлять/удалять студентов из групп.
- Управляет курсами и расписанием.

## API Report (Swagger: `/swagger/`)

### Авторизация (OTP на Email)
- `POST /api/users/register/` - Регистрация нового пользователя.
- `POST /api/users/request_otp/` - Запрос OTP на имейл.
- `POST /api/users/verify_otp/` - Проверка OTP и получение Token.

### Образование
- `GET /api/courses/` - Список курсов (Все).
- `GET /api/groups/` - Список групп (Зависит от роли).
- `POST /api/groups/` - Создание группы (Только Куратор).
- `GET /api/schedules/` - Расписание (Зависит от роли).
- `GET /api/grades/` - Оценки (Зависит от роли).
- `POST /api/grades/` - Выставление оценки (Преподаватель/Куратор).
